# Background
Before now, all flows were hand-generated. They're generally small sequences organized into group blocks, and those sequence nodes are hand laid out to have clear alignment, visual presentation of branching, etc. There's a lot of thought put into how nodes are laid out. We'll ultimately want to see if we can create rules for an agent to intelligently layout nodes after we implement changes to the json.

# Task
Reading the file generated above (docs/plans/2026-02-16-flow-layout-analysis.md), reason about how to construct a Claude skill that lives within this project that can relayout a subset of nodered.json to abide by these rules and be maintanable by a human. The skill should start off by looking at `helper-scripts/summarize-nodered-flows-diff.sh nodered-last-downloaded.json nodered.json` so it knows what has changed on disk that we haven't deployed yet. It will focus on reformatting all of the nodes/groups created or modified in that diff, but also will shift other nodes and groups AS NEEDED. In general, it should be as conservative as possible with the nodes: Definitely position created nodes/groups, but only move other nodes/groups as necessary. And when possible, it should strictly translate other nodes by the same amount such that if there's e.g. a large branch off of a node you changed, and that branch has a whole bunch of carefully human-decided layout to it, it should maintain that same layout relative to itself; the whole branch changes. Things like that. Shift groups around as needed too, but try to keep them roughly in the same arrangement, just try to resolve overlaps.
