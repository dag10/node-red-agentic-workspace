# Background
Before now, all flows were hand-generated. They're generally small sequences organized into group blocks, and those sequence nodes are hand laid out to have clear alignment, visual presentation of branching, etc. There's a lot of thought put into how nodes are laid out. We'll ultimately want to see if we can create rules for an agent to intelligently layout nodes after we implement changes to the json.
# Task
Look at some specific flows that have a particularly nice layout: "bedtime", "tablets", "switches", and "vacation mode". Within each flow, for each group, identify the layout of all the nodes in that group and how they're thoughtfully organized to convey the intent and flow within the group according to its purpose. Once you've done that for each group, also look at the bounds of each group itself within the entire flow, and note all of your observations about spacing, location and proximity, alignment, etc. Finally, once you've done this for each group and their nodes for a given flow, write down all of your notes about the implied best practices for that flow. Be very thorough because this will ultimately result in creating some detailed instructions for future subagents to be able to intelligently reformat a flow. Save all of this information (all flows) in one markdown file in plans.
